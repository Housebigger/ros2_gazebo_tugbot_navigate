# Code of Conduct

## Our Pledge

We want this project to be a respectful, technically serious, and collaborative engineering space.

Participants are expected to:

- communicate professionally
- focus on technical facts and evidence
- welcome good-faith questions and corrections
- document reasoning when making impactful changes
- avoid dismissive or hostile behavior

## Our Standards

Examples of behavior that contributes to a positive environment:

- giving constructive technical feedback
- sharing reproducible evidence
- admitting uncertainty and testing assumptions
- helping others understand architecture boundaries and validation results
- improving documentation when discovering a pitfall

Examples of unacceptable behavior include:

- personal attacks, insults, or harassment
- repeated bad-faith argumentation
- intentionally misleading reports about test or runtime results
- publishing others' private information
- disruptive behavior that blocks normal collaboration

## Enforcement Responsibilities

Project maintainers are responsible for clarifying and enforcing acceptable behavior.

They may remove, edit, or reject comments, commits, issues, documentation, or other contributions that violate this Code of Conduct.

## Scope

This Code of Conduct applies within project spaces and in public spaces when an individual is representing the project.

## Enforcement

To report unacceptable behavior, contact the maintainers through the repository's normal private coordination channel.
