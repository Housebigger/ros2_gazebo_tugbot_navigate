## Design decisions

Future Phase124 may do exactly this:

- launch visible Gazebo/RViz/SLAM/Nav2;
- reproduce the Phase120 managed-active readiness wait, strict preflight, and explicit inner-ingress goal;
- require the Phase122 handoff-ready gate: `handoff_allowed=true`, Nav2 idle, scan/TF/costmap fresh, and robot pose near ingress;
- start `maze_explorer` only after handoff is ready;
- start `maze_explorer` with `max_goals=1`;
- allow first exploration goal only;
- require the first dispatch to be generated by the current `maze_explorer` algorithm;
- stop immediately after the first terminal outcome.

Future Phase124 must not:

- manually specify Goal1;
- inject a hand-authored pose;
- use fallback goal as success;
- use terminal goal as success;
- use exit goal as success;
- continue autonomous exploration after the first goal;
- tune Nav2/MPPI/controller/config;
- change exploration strategy;
- change branch scoring;
- change centerline gate;
- change fallback;
- change terminal acceptance.
